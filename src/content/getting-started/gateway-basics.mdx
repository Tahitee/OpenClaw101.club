# Gateway 基础

Gateway 是 OpenClaw 的核心组件，负责消息路由、会话管理和技能调度。本篇将深入讲解 Gateway 的工作原理和基本操作。

## 什么是 Gateway

Gateway 是 OpenClaw 的守护进程，它持续运行在后台，接收来自各个渠道的消息，调用 AI 模型处理，并将结果返回给用户。

## 基本操作

包括启动、停止、重启 Gateway，以及如何查看运行状态和日志。掌握这些基本操作是使用 OpenClaw 的基础。

## 配置 Gateway

Gateway 的行为可以通过配置文件来控制。我们会介绍最常用的配置项和推荐的初始设置。
